## O que NÃO requer licença comercial

| Caso | Exemplo |
|---|---|
| Self-host para uso próprio da associação | Associação instala o sistema nos próprios servidores para gerenciar seus membros |

| Contribuições ao projeto | Pull request, fork com patches publicados sob AGPL |
| Fork AGPL-compliant | Fork mantido publicamente com todo código-fonte disponível sob AGPL |

## O que REQUER licença comercial

Uma licença comercial é obrigatória nos três casos abaixo:

### Caso 1 — Embeber em produto sem publicar código

Integrar o canna-br (ou parte dele) em outro software comercial distribuído ou oferecido como serviço, **sem disponibilizar o código-fonte completo** sob AGPL.

**Exemplo concreto:** Empresa de software de farmácias inclui o módulo de rastreabilidade do canna-br no seu produto SaaS proprietário.

### Caso 2 — Serviço gerenciado para terceiros (integrador parasita)

Hospedar e operar o canna-br como serviço para **outras organizações** sem ser a própria organização usuária — mesmo sem modificar o código.

**Exemplo concreto:** Consultoria local hospeda o canna-br para 10 associações clientes a R$ 500/mês sem contribuir de volta ao projeto e sem publicar código-fonte.

> **Nota:** Hospedar para uso exclusivo da própria organização **não** se enquadra aqui.

### Caso 3 — Integração comercial com certeza jurídica

Integrar o canna-br via API, eventos ou MCP em sistema proprietário e precisar de **certeza jurídica explícita** de que a integração não configura obra derivada AGPL.

**Exemplo concreto:** ERP regional integra rastreabilidade do canna-br via API REST para clientes seus; precisa de garantia contratual de que o ERP não vira "obra derivada" obrigada a abrir código.
