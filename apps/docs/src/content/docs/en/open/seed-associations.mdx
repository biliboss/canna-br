## We are looking for 5 associations for a seed pilot period.

This is not a sale. It is an invitation to co-build a system alongside the people who will use it.

A patient association (Brazilian regulatory entity — non-profit licensed to import, cultivate, or dispense medical cannabis to its members) is the core operational unit that canna-br is built for. Brazil currently has 400+ associations mapped; ~315 addressable (SAM) operating under judicial habeas corpus, with RDC 1.014/2026 (the first formal administrative regime for these entities) coming into force August 2026.

canna-br is functional — LGPD-by-design, traceability, dispensation, SNGPC in development (mock currently; awaiting ANVISA XSD schema) — but has not been used in real production yet. We want associations willing to go first, knowing that, and help us sharpen the product before we open to the broader market.

## Data sovereignty commitments

Your members' data is yours. This is not rhetoric — it is architecture.

- Per-member encryption: each member has their own encryption key (random DEK). See the model at [LGPD — Encryption](/build/compliance/).
- Real crypto-deletion: when a member requests deletion (Art. 18 IV LGPD), the key is destroyed. Encrypted data becomes unrecoverable noise. The system never stores CPF (Brazilian tax ID) in plaintext.
- Self-host available: if you prefer running on the association's own server, the system is fully self-hostable. No dependency on our infrastructure.
- No lock-in: full export in open formats at any time.
- Auditable code: any lawyer, technician, or member can review the code that processes data. Public repository on GitHub.
