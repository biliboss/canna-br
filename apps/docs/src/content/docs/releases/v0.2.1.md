## Architecture changes

Domain Kernel intacto desde v0.2.0b ([ADR-001](/adr/0001-domain-kernel-emmett/) spike gate passed). Surface pivot consolidado em [ADR-002](/adr/0002-mcp-first-surface/).

> Operação rotineira de dispensação cabe muito bem em fluxo conversacional. MCP Apps spec final (jan 2026) torna o caminho viável. Open WebUI v0.9.6+ tem suporte MCP nativo + OAuth 2.1 — chat UI gratuita. Time pequeno, cada hora gasta em Next.js admin é hora não gasta em MCP tools. Para v0.2.1 até v1.0 o canna-oss **não terá admin Next.js standalone** — toda interação humana acontece via MCP server + MCP Apps inline + REST `apps/api` (Nível-4 critical commands com TOTP).

## Breaking changes

- **Admin Next.js standalone — descartado.** Estava no roadmap original v0.2.1 como interface canônica. Movido para [Ideas Park no roadmap](/roadmap/#ideas-park). Caso surja associação que não consegue operar via chat, avaliar PWA fallback naquele momento — não pré-construir.
- Toda interação humana com o sistema agora acontece via Open WebUI chat + MCP Apps inline. REST `apps/api` continua exposto mas só para Nível-4 critical commands (crypto deletion, role change, recall, SNGPC produção) com TOTP.

## Migration notes

Nenhuma. Sem deploy de produção anterior, sem schema migration, sem upgrade path. Pré-piloto.
