## Procuramos 5 associações para um período inicial de teste.

Isso não é uma venda. É um convite para co-construir um sistema junto com quem vai usá-lo.

O canna-br está funcional — LGPD, rastreabilidade, dispensação, SNGPC em desenvolvimento (mock atual; aguarda XSD ANVISA) — mas não foi usado em produção real ainda. Queremos associações que topem ser as primeiras, sabendo disso, e que nos ajudem a afinar o produto antes de abrirmos para o mercado amplo.

---

## Quem buscamos

- Associação já operante, com dispensação ativa de membros-pacientes
- Responsável Técnico (farmacêutico) identificado na operação
- Diretoria disposta a participar de co-design: dar feedback real, não validar o que já está pronto
- Confortável com software em fase alpha — bugs acontecem, e o combinado é reportar, não reclamar
- OK com o modelo OSS (AGPL-3.0): o código é público e auditável por qualquer pessoa

---

## O que você ganha

- Sistema completo sem custo durante o período seed: membros, prescrições, dispensação, rastreabilidade, audit trail · cultivo + financeiro + lab no roadmap v1.0 (2028)
- Suporte direto com Gabriel — sem ticket, sem fila, contato pelo Discord ou email
- [LGPD by design](/build/compliance/): criptografia AES-256-GCM por membro, crypto-deletion Art. 18 IV garantida
- [SNGPC mock — pronto para integração quando ANVISA publicar XSD](/idea/regulation/): adapter XML implementado, aguarda schema oficial ANVISA
- Escolha entre self-host (você controla o servidor) ou hosted em `canna-br.fonsecagabriel.com.br`
- Exportação completa dos seus dados a qualquer momento, sem custo, sem burocracia
- Citação pública como parceira fundadora quando o projeto abrir para o mercado

---

## O que pedimos

- Feedback semanal: uma mensagem curta sobre o que funcionou, o que não funcionou, o que está faltando
- Reportar bugs no GitHub Issues quando encontrar (com contexto suficiente para reproduzir)
- Aceitar que nem tudo estará pronto: algumas telas existem, outras são mocks
- Participar de uma videochamada mensal de 30 min com Gabriel (opcional, mas valorizado)
- Citação como parceira fundadora no `/community` quando o projeto for público (combinado antes)

---

## Compromissos de soberania

Os dados dos seus membros são seus. Isso não é retórica — é arquitetura.

- Criptografia por membro: cada membro tem sua própria chave de criptografia (DEK aleatória). Veja o modelo em [LGPD — Criptografia](/build/compliance/).
- Crypto-deletion real: quando um membro solicita eliminação (Art. 18 IV LGPD), a chave é destruída. Os dados cifrados se tornam ruído irrecuperável. O sistema não guarda CPF em claro em nenhum momento.
- Self-host disponível: se preferir rodar no servidor da associação, o sistema é integralmente self-hostável. Sem dependência de infraestrutura nossa.
- Sem lock-in: exportação completa em formatos abertos a qualquer momento.
- Código auditável: qualquer advogado, técnico ou membro pode revisar o código que processa os dados. Repositório público no GitHub.
