# AGPL-3.0 é decisão de confiança, não técnica

**Assertiva:** a escolha de AGPL-3.0 (com dual-license via CLA) é de negócio e
confiança, não técnica: dados sensíveis de saúde em setor regulado exigem que o
código que os processa seja auditável e não-capturável.

Esse moat só faz sentido porque a [compliance-engine](compliance-engine.md) e o
[domain-kernel](domain-kernel.md) são abertos e verificáveis ponta-a-ponta.
